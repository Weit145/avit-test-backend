### 4. Структура проекта

Вся основная кодовая база находится в папке `app/`:

```
app/
├── api/               # эндпоинты и схемы
├── service/           # бизнес-логика
├── storage/           # репозитории и ORM-модели
├── core/              # конфигурация, логирование, исключения
├── infrastructure/    # JWT, база данных, внешние сервисы
└── mappers/           # преобразователи моделей
```

Корневая директория также включает:

- **tests/**  
  - *unit/* — юнит-тесты для сервиса
  - *integration/* — интеграционные тесты.  
- **migrations/** — директория с миграциями Alembic, автоматически генерируемыми и управляемыми.


### 5. Технологический стек

| Компонент          | Технология                |
|--------------------|---------------------------|
| Фреймворк          | FastAPI                   |
| ORM                | SQLAlchemy 2.x (асинхронный) |
| Миграции           | Alembic                   |
| Валидация данных   | Pydantic v2               |
| Аутентификация     | JWT (python-jose)         |
| База данных        | PostgreSQL (или SQLite для тестов) |
| Тестирование       | pytest, pytest-asyncio, httpx |
| Логирование        | стандартный logging       |
| Управление конфигурацией | pydantic-settings   |

### 7. Миграции

Миграции генерируются автоматически с помощью Alembic при изменении моделей SQLAlchemy.  
- Используется асинхронный режим работы с БД.  


### 8. Развёртывание и запуск

Проект может быть запущен локально с помощью Docker Compose, либо напрямую через uvicorn.  
Предусмотрены:

- `docker-compose.yml` — поднимает PostgreSQL, Redis (если требуется) и приложение.  
- `.env.example` — шаблон для переменных окружения.  
- `Makefile` — удобные команды для запуска, миграций, тестов.
